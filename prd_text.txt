Executive SummaryVeridian is a production-grade, AI-native Trust Infrastructure Platform that detects, traces, contextualizes, and actively counters misinformation across text, images, audio, and video — in real time — with first-class support for 22 Indian languages. It is not a detection API wrapper. It is a six-layer reasoning engine with proprietary claim-graph infrastructure, provenance tracing, and a counter-narrative generation system that injects verified truth back into the same channels where misinformation spreads. India alone saw over 500 million fake news shares during recent election cycles. WhatsApp — with 500M+ Indian users — is the primary vector, not Twitter or Facebook. This means misinformation spreads as blurry screenshots, re-shared voice notes, and recycled videos with no URL to fact-check against.2.2 Why Existing Tools FailManual fact-checking (AltNews, FactChecker.in) cannot scale — they check hundreds of claims per month against millions that need verificationEnglish-only detection models (OpenAI detectors, Hive Moderation) fail on Hindi, Tamil, Telugu, Bengali, and 18 other Indian languagesDetection-only products give a score with no action — they do not tell users what to believe, just what might be falseNo existing tool traces the temporal origin of recycled content — the biggest Indian misinformation patternWhatsApp-native pipelines that can receive and respond to forwarded media do not exist as commercial products2.3 The Veridian Thesis Mission StatementShift misinformation response from passive detection to active, real-time trust restoration — by building the trust layer that every digital platform, government body, and fact-checking newsroom needs underneath their workflow.3. System ArchitectureVeridian is built as six composable, independently scalable layers. Each layer has a defined input contract, output schema, and failure mode. Layers communicate via an internal message bus (Redis Streams) and a shared object store (S3-compatible). Design PrincipleNo layer is a black box. Every decision — detection score, retrieved evidence, verdict — is traceable, logged, and explainable. This is mandatory for legal defensibility and journalist trust.Layer 1 — Ingestion &amp; Media Decomposition1.1 Entry PointsWhatsApp Cloud API webhook (receives forwarded text, images, audio, video, documents)Telegram Bot API (secondary channel, same capabilities)REST API — POST /v1/analyze with multipart media uploadBrowser Extension — captures active tab content, embeds, and on-page mediaBatch CSV/JSON ingestion endpoint for newsroom/enterprise bulk workflows1.2 Media Decomposer PipelineEvery inbound artifact is atomized into modality-specific units before entering detection:Video: extract keyframes at 1fps via ffmpeg, separate audio track as WAV, strip metadata to JSONImage: normalize to PNG, extract EXIF/metadata, generate perceptual hash (pHash) for deduplicationAudio: convert to 16kHz mono WAV, extract speaker segments via pyannote.audioText/URL: scrape with Playwright (JavaScript-rendered), extract publish date, author, canonical URL, language detection via fastTextWhatsApp Forward: extract forward_count field (&gt;5 forwards = elevated risk signal), capture media even without URL1.3 Deduplication &amp; Cache LayerpHash-based near-duplicate detection prevents re-analyzing the same image/video. Results are cached in Redis with a 72-hour TTL. Cache hit serves result in &lt;50ms. This is critical for viral misinformation where the same asset is forwarded millions of times.Key Tools: yt-dlp, ffmpeg, Playwright, pyannote.audio, fastText, RedisLayer 2 — Multimodal Detection EngineFour independent sub-pipelines run in parallel via Celery workers. Results are aggregated into a unified MultimodalDetectionReport with per-modality confidence scores.2.1 Text DetectionBinoculars (2024): Zero-shot AI-generated text detection using perplexity ratio between two observer/performer LLMs. No fine-tuning required. SOTA on cross-domain detection without training data leakage.MuRIL (Google): Multilingual Representations for Indian Languages. Detects semantic manipulation, paraphrase-based fake news, and out-of-context caption misuse across 17 Indian languages.Claim-level credibility pre-scorer: Before full verification, lightweight BERT classifier estimates claim check-worthiness to prioritize expensive LLM calls.2.2 Image DetectionELA (Error Level Analysis): JPEG re-compression artifact analysis detects manual splicing and local edits. Implemented via imageio + custom ELA kernel.DIRE (Diffusion Reconstruction Error, 2023): Reconstructs the image using a pretrained diffusion model and measures pixel-level divergence. Detects Midjourney, DALL-E 3, Stable Diffusion outputs that fool CLIP-based detectors.Out-of-context image detection: CLIP embeddings compare image content against claimed caption/context. High cosine divergence = misuse signal.Metadata forensics: Missing EXIF, implausible creation timestamps, GPS coordinate mismatches all contribute to a composite forgery score.2.3 Audio DetectionRawNet2: End-to-end voice spoof/deepfake detection trained on ASVspoof dataset. Detects TTS synthesis and voice conversion attacks at the waveform level.Resemblyzer: Generates 256-dim d-vector speaker embeddings. Compares claimed speaker identity against a curated VoicePrint Database of 500+ Indian public figures (politicians, officials, journalists).Acoustic inconsistency detection: Background noise profile analysis, unnatural prosody scoring, codec artifact fingerprinting.2.4 Video DetectionFaceForensics++ (EfficientNet-B4): Face-crop based deepfake detection. Detects FaceSwap, DeepFakes, Face2Face, and NeuralTextures manipulations.Temporal consistency analysis: Real videos maintain consistent noise profiles, lighting gradients, and compression artifacts across frames. Composited or AI-generated videos exhibit frame-level discontinuities detectable via optical flow anomaly scoring.Lip-sync mismatch: SyncNet-based audio-visual sync scoring. Dubbed/re-lipped videos produce characteristic sync delays.Scene boundary manipulation: Detects cut-and-paste editing where authentic footage is spliced with fabricated segments.Key Tools: Binoculars, MuRIL, DIRE, RawNet2, Resemblyzer, FaceForensics++, SyncNet, Celery + RedisLayer 3 — Claim Extraction &amp; Fact VerificationThis layer transforms raw media into structured, falsifiable claims and verifies each claim against a curated evidence corpus. This is the most technically differentiated layer of Veridian.3.1 Claim Extraction AgentA LangGraph-orchestrated agent pipeline processes each media unit:Transcription: Whisper large-v3 for audio/video, OCR via PaddleOCR for text in imagesTranslation: IndicTrans2 normalizes all content to English for unified claim extraction, preserving a parallel Indian-language version for outputClaim decomposition: LLM with structured output extracts atomic falsifiable claims (e.g., &apos;Claim: Government announced scheme X on date Y&apos;) with entity tagsCheck-worthiness scoring: Each claim scored 0-1 by ClaimBuster model. Only claims above threshold 0.65 proceed to full verification3.2 Evidence Retrieval (RAG Pipeline)A hybrid retrieval system queries multiple authoritative sources in parallel:Qdrant vector database indexes: PIB press releases, WHO advisories, MoHFW circulars, AltNews archive, FactChecker.in, Snopes, Reuters Fact Check — updated daily via scraping pipelineWikidata SPARQL queries for entity-level fact verification (dates, positions, statistics, official statements)Tavily real-time web search for claims requiring current information beyond the static corpusIndian government open data portals (data.gov.in) for statistical claims3.3 Evidence-Grounded Verdict GenerationThe LLM receives: original claim + retrieved evidence snippets (with source URLs and dates) + structured JSON schema. It is explicitly instructed: do not reason from training data, reason only from provided evidence. This eliminates hallucination from the verification loop.Output schema: { verdict: TRUE|FALSE|MISLEADING|UNVERIFIABLE, confidence: 0-1, evidence_used: [...], reasoning: string, caveat: string }Key Tools: LangGraph, Qdrant, Haystack, Whisper large-v3, PaddleOCR, IndicTrans2, ClaimBuster, Tavily APILayer 4 — Temporal &amp; Provenance TracingThis is Veridian&apos;s most unique capability and the hardest to replicate. The majority of Indian misinformation involves recycled authentic content reframed as current — a 2019 video presented as today&apos;s breaking news.4.1 Reverse Provenance SearchGoogle Vision API reverse image search to find earliest known online occurrenceTinEye reverse image search as secondary source (different crawl corpus)YouTube video fingerprinting via Content ID-style audio/visual hash matching against a pre-indexed archive of viral Indian news contentWayback Machine CDX API to check if URLs predate their claimed publish date4.2 Coordinated Inauthentic Behavior DetectionVeridian maintains a real-time Claim Graph in Neo4j. Every verified or flagged claim is stored as a node with language, source, timestamp, and embedding vector. Edges represent semantic similarity (cosine &gt; 0.85 via cross-lingual embedding). The graph enables:Detecting coordinated campaigns: same claim appearing in Hindi, Tamil, Marathi within 6 hours = likely coordinated amplificationTracing superspreader nodes: which sources/accounts originate claims that later go viralElection season monitoring: automated alerts when claim velocity exceeds baseline thresholds4.3 Cross-Lingual Claim NormalizationIndicTrans2 translates all incoming claims to English before embedding and graph insertion. This ensures that semantically identical claims in different languages are recognized as the same claim — something no existing Indian fact-check platform does.Key Tools: Neo4j, IndicTrans2, Google Vision API, TinEye, Wayback Machine API, Qdrant (claim similarity search)Layer 5 — Counter-Narrative Generation EngineDetection is table stakes. The Trust Receipt is Veridian&apos;s product-defining feature. Instead of simply flagging content, Veridian generates a structured, cited, shareable debunk in the same language as the incoming query.5.1 Trust Receipt SchemaEvery Trust Receipt is a structured JSON object rendered as a shareable card:CLAIM: The extracted falsifiable claim (in original language)VERDICT: TRUE / FALSE / MISLEADING / UNVERIFIABLE with confidence percentageEVIDENCE: 2-3 bullet points citing authoritative sources with URLs and publication datesORIGIN: When and where this content first appeared (from Layer 4 provenance tracing)CONTEXT: Why this claim tends to circulate (elections, budget season, communal tension events)RESPONSE: A 3-5 sentence paragraph in the user&apos;s language, suitable for forwarding back into the same WhatsApp group, with citations embeddedQR CODE: Links to the full Veridian report page for that claim5.2 Multilingual Output GenerationThe response paragraph is generated in English and machine-translated to the target language via IndicTrans2, then post-edited by a language-specific fluency model (fine-tuned MuRIL) to ensure grammatical correctness in Devanagari, Dravidian, and other script families.5.3 Tone CalibrationTrust Receipts are tuned to be informative, not condescending. Research on misinformation correction shows that accusatory corrections backfire. The LLM prompt enforces a neutral, educational tone that corrects facts without shaming the person who shared them.Key Tools: LangGraph, IndicTrans2, MuRIL (fluency post-edit), QR code generation (qrcode library), Jinja2 templatesLayer 6 — Delivery Surfaces6.1 WhatsApp BotPrimary delivery channel for Indian end consumers. User forwards any content to the Veridian WhatsApp number. The bot responds with a Trust Receipt image card + short text summary in the detected language within 10 seconds for text/images, 30 seconds for video. Built on WhatsApp Business Cloud API with Twilio as fallback.6.2 Telegram BotIdentical functionality to WhatsApp bot. Telegram&apos;s bot API allows richer formatting, inline buttons, and file attachments. Secondary channel for users outside WhatsApp ecosystem.6.3 Chrome/Firefox Browser ExtensionPassive protection layer for desktop users. Scans page text and embedded media against the Veridian API on page load. Displays a subtle trust indicator (green shield / amber warning / red alert) on embedded images, videos, and quoted text blocks. Full Trust Receipt popup on click. No user action required for basic protection.6.4 Journalist &amp; Newsroom DashboardWeb application (Next.js) for power users: bulk claim submission, claim graph visualization (Neo4j rendered via D3.js), trending misinformation heatmap by language and geography, export to PDF/CSV for publication fact-check sections, API key management, webhook configuration.6.5 REST API (Enterprise)Full programmatic access for platform trust &amp; safety teams and government fact-check cells. Rate-limited by tier. Supports async webhook callbacks for large media files. Documented via OpenAPI 3.1 spec with Postman collection.Key Tools: WhatsApp Cloud API, Telegram Bot API, Plasmo (browser extension framework), Next.js, D3.js, FastAPI5.
